### 1) 첨부 파일 내용 미반영 ("양식을 알지 못해")
- **증거**: 실사용 로그에서 첨부 직후 턴은 실제 데이터(Celltrion Branchburg, ZAQMR0060 등) 반영

- **원인**: `index.html send()` — 첨부 텍스트(fileText)가 **현재 턴 message에만** 합쳐지고,
  대화 저장(`r.messages.push({role,text})`)과 히스토리(`map(m=>m.text)`)에는 **빠짐**.
  → 다음 턴부터 AI가 첨부 내용을 완전히 잊음.
- **수정**: 메시지 저장 시 `att`(파일명+텍스트, 파일당 6000자·최대 3개 캡) 보존 +
  신설 `buildChatHistory()`가 히스토리에 첨부 텍스트 주입(4000자 캡, **마지막=현재 메시지는
  제외**해 현재 턴 전문 전달과 중복 방지).

## 추가 수정 (같은 날, v110) — Drive 링크 429 "Request too large"

### 증상 (실사용 스샷)
Drive 폴더 링크 + "MS Word로 만들어줘" → `OpenAI 429: Request too large for gpt-4o —
TPM Limit 30000, Requested 41711`.

### 진단
**링크 읽기는 성공**(폴더 내용이 실제 주입돼 41,711 토큰까지 커짐)했으나, 요청 크기가
사용자 OpenAI 조직의 gpt-4o 분당 토큰 한도(TPM 30,000)를 초과. 한글 문서는 자당 토큰이
높아 기존 Drive 예산(50,000자)이 과대했고, 신설된 첨부 히스토리 주입까지 겹침.

### 수정 (3중 방어)
1. Drive 컨텍스트 예산: 총 50,000자→22,000자, 파일 20→12개 (`lib/drive-utils.js`)
2. 히스토리 가드: 서버 `trimHistoryByChars`(최근 우선 24,000자) +
   클라 `buildChatHistory` 첨부 주입 총 예산 8,000자(최신 우선); 2차 Drive 캡 60K→28K자
3. **429 자가 치유**: gpt-4o 429 시 TPM 한도가 넉넉한 gpt-4o-mini 로 자동 재시도,
   그래도 초과면 히스토리 4개로 줄여 최종 재시도 (`timings.tpmFallback` 기록)

### UI (사용자 요청)
- Stella GPT 테마 토글(🌙/☀️): 무채색·저채도(hover 시 진해짐)
