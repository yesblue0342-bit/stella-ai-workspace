# 목표: Stella GPT 검색 기능 완전 수정 (gpt.이후.com, index.html)

## 현재 증상 (사용자 실측)
1. 사이드바 검색창에 단어를 입력하고 돋보기 버튼을 눌러도 결과창이 갱신되지 않는다.
2. 결과창에 이전 검색어의 "검색 결과 없음" 메시지가 그대로 남아 있다 (입력값과 결과 라벨 불일치: "주간" 입력 시 "프롬프크" 결과 표시).
3. 단, 사이드바 카테고리/채팅 목록에는 데이터가 정상적으로 보인다 → 데이터 로드는 정상, 결과 렌더링이 문제.
4. 검색이 노트와 채팅 "전체"(제목+본문 메시지 내용)를 대상으로 동작해야 하는데 그렇지 않다.

## 할 일
1. index.html 의 검색 입력 처리/결과 렌더링 코드를 정독한다.
2. 결과창이 안 갱신되는 원인을 찾아 고친다 (이전 결과/라벨이 남는 상태 버그, 입력값과 렌더 결과 불일치 포함).
3. 검색을 입력 즉시 실시간으로 동작하게 한다 (input 이벤트로 필터링, 돋보기 버튼도 계속 동작).
4. 검색 대상 범위를 확대한다: 채팅 제목 + 채팅 메시지 본문 + 노트 제목 + 노트 본문 전체. 대소문자 무시, 한글 부분일치.
5. 검색 결과를 카테고리(채팅/노트)별로 구분해 표시하고, 0건일 때만 정확한 현재 검색어로 "결과 없음"을 보여준다.
6. service worker 캐시 버전을 한 단계 올린다.
